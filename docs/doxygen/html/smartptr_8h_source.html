101  /// @brief This is used to deduce at compile if a specific class has built reference counting or
102  /// specialized reference counting.
103  /// @details Any class that provides TypePointedTo* GetReferenceCountPointer(), Whole GetReferenceCount(),
104  /// Whole IncrementReferenceCount() and Whole DecrementReferenceCount() can be used as a reference
105  /// counter. If it provides these then a specialization of ReferenceCountTraits should be
106  /// implemented for a given class and CountedPtr will use the any type defined by
107  /// ReferenceCountTraits<T>::ManagedType as the counter. @n @n
108  /// The Mezzanine provides an Reference count class that can be used with any type at the cost of
109  /// extra dereferences. Some type (Scripts) have their own internal reference count that when used
110  /// will increase the locality (since the reference count is part of the managed object) and reduce
111  /// dereferences to exactly one. Since the CountedPtr is the size of a native pointer if it is used
112  /// with an internal reference count, the only overhead should be the counting itself.
113  /// @todo Benchmark CountedPtr, all this theory is nice, but numbers really matter. It should be compared to std::shared_ptr and should be faster since it is intrusive and provides no thread safety.

168  /// @brief Every class that does not implement its own reference count gets the default one.
169  /// @details The Default reference count is not thread-safe, and requires that every dereferencing
170  /// of the smart pointer is actually a double dereference. The double dereference is caused because
171  /// the reference counter has to store a native pointer to the managed object.
